## Amazon EC2とは

Amazon EC2 (Amazon Elastic Compute Cloud)は、AWSが提供するクラウド上でOS付きの仮想マシン（VM）を作成できるサービス。インターネット上に自分専用のパソコン（サーバ）を立ち上げるような感覚で利用できる。

EC2は、**IaaS (Infrastructure as a Service)** に分類されるサービスで、インフラストラクチャをクラウド上で提供する。IaaSの中でも最も有名で広く用いられており、AWSの主要サービスの一つ。

## IaaSとは？

IaaS (Infrastructure as a Service) は、クラウドサービスの一形態で、仮想化されたコンピュータリソース（サーバ、ストレージ、ネットワーク）を提供する。ユーザーはハードウェアの管理をクラウドプロバイダーに任せ、必要なときに必要なリソースをオンデマンドで利用できる。

## SaaS, PaaS, IaaSの比較

| 項目               | IaaS                      | PaaS                      | SaaS                        |
|--------------------|---------------------------|---------------------------|-----------------------------|
| データ             | ユーザー管理              | ユーザー管理              | プロバイダー管理            |
| アプリケーション   | ユーザー管理              | ユーザー管理              | プロバイダー管理            |
| ミドルウェア       | ユーザー管理              | プロバイダー管理          | プロバイダー管理            |
| OS                 | プロバイダー管理          | プロバイダー管理          | プロバイダー管理            |
| 仮想化ソフトウェア | プロバイダー管理          | プロバイダー管理          | プロバイダー管理            |
| ハードウェア       | プロバイダー管理          | プロバイダー管理          | プロバイダー管理            |
※ただし、EC2は一部ミドルウェアまでカバーしてくれる

### 各サービスの特徴

### IaaS (Infrastructure as a Service)
- **概要**: 仮想化されたインフラストラクチャ（サーバ、ストレージ、ネットワーク）を提供。
- **利点**: 高い柔軟性とカスタマイズ性、オンデマンドでリソースを利用可能。
- **ユーザーの役割**: OSやアプリケーションのインストールと管理を行う。
- **例**: AWS EC2、Google Compute Engine

### PaaS (Platform as a Service)
- **概要**: アプリケーションの開発・実行環境を提供。
- **利点**: 開発の迅速化、インフラ管理の簡便化。
- **ユーザーの役割**: アプリケーションの開発とデプロイを行う。
- **例**: AWS Elastic Beanstalk、Google App Engine

### SaaS (Software as a Service)
- **概要**: 完成されたソフトウェアをサービスとして提供。
- **利点**: 即時利用可能、ソフトウェアのインストールや管理が不要。
- **ユーザーの役割**: ソフトウェアを利用するだけで、インフラやアプリケーションの管理は不要。
- **例**: Google Workspace、Microsoft Office 365

IaaSはクラウドサービスの基盤を提供し、ユーザーは高度なカスタマイズが可能。PaaSは開発環境を提供し、開発者の負担を軽減。SaaSは完成されたアプリケーションを提供し、エンドユーザーが即時に利用可能。各サービスの選択は、ユーザーのニーズとスキルに応じて行うことが重要。

## EC2の構造
EC2は以下の構成要素から成り立っている。

| 名称                | 通常のPCにおける役割                     |
|---------------------|------------------------------------------|
| AMI (OS)            | OS (Linux, Windows等) + 一部のミドルウェア|
| インスタンスタイプ  | CPU + メモリ (+ NVMeストレージ + GPU)    |

AMI (Amazon Machine Images)はインスタンス起動時に実行されるOSイメージ。OSだけでなくミドルウェアがインストールされたAMIもある。

## Debianとは？

### 概要
Debianは、フリーで**オープンソース**のLinuxディストリビューションの一つ。多くの派生ディストリビューションの基盤となっている。プロジェクトは1993年にIan Murdockによって設立された。

### 特徴
- **安定性**: 高い安定性を重視しており、サーバー環境で広く使用されている。
- **パッケージ管理**: APT (Advanced Package Tool) を使用し、豊富なソフトウェアパッケージを提供。
- **フリーソフトウェア**: デフォルトでフリーソフトウェアのみを提供し、プロプライエタリソフトウェアはユーザーが別途設定する必要がある。
- **マルチプラットフォーム**: 多くのハードウェアアーキテクチャをサポート（x86, ARM, MIPSなど）。

### リリース
- **安定版 (Stable)**: 生産環境での使用に適しており、リリースごとに徹底的なテストが行われる。
- **テスト版 (Testing)**: 次の安定版リリース候補。比較的新しいソフトウェアを含むが、安定版ほどのテストはされていない。
- **不安定版 (Unstable)**: 開発の最前線。最新の機能を試すことができるが、バグが含まれる可能性が高い。

### 利点
- **高い安定性とセキュリティ**: 長期間にわたるサポートとセキュリティアップデート。
- **幅広いソフトウェア**: 公式リポジトリには多数のソフトウェアパッケージが存在し、簡単にインストール・管理が可能。
- **コミュニティサポート**: 世界中のユーザーと開発者による活発なコミュニティ。

### 派生ディストリビューション
Debianは多くの人気ディストリビューションの基盤となっている。
- **Ubuntu**: Canonicalが開発。デスクトップ用途で広く使用される。
- **Linux Mint**: Ubuntuをベースにした使いやすさ重視のディストリビューション。
- **Raspberry Pi OS**: Raspberry Pi向けに最適化されたDebianベースのOS。

### 利用例
- **サーバー環境**: Webサーバー、メールサーバー、ファイルサーバーなどのバックエンドシステムで使用。
- **デスクトップ環境**: 個人用のPCや開発環境として利用。
- **教育用途**: 教育機関でのコンピュータ教育や研究用途に使用。

## Red Hatとは？

### 概要
Red Hatは、オープンソースソフトウェアの開発および提供を行う**企業**であり、その代表的な**製品**がRed Hat Enterprise Linux (RHEL)。RHELは、企業向けに最適化された商用Linuxディストリビューション。

### 特徴
- **企業向け**: 企業のITインフラストラクチャのニーズに対応し、高い安定性とセキュリティを提供。
- **サポート**: 企業向けに有償のサポートとコンサルティングサービスを提供。
- **認証**: 多くの企業の認証を取得しており、商用環境での信頼性が高い。
- **パートナーシップ**: 多くのハードウェアベンダーやソフトウェアプロバイダーと提携している。

### 主な製品とサービス
- **Red Hat Enterprise Linux (RHEL)**: 安定性、セキュリティ、パフォーマンスを重視したエンタープライズ向けのLinuxディストリビューション。
- **OpenShift**: コンテナオーケストレーションプラットフォーム。Kubernetesを基盤としている。
- **Ansible**: IT自動化ツール。プロビジョニング、構成管理、アプリケーションデプロイに使用。
- **Red Hat Satellite**: システム管理とライフサイクル管理のためのツール。

### 利点
- **信頼性**: 長期的なサポートと定期的なセキュリティアップデートにより、商用環境での高い信頼性を提供。
- **サポートとトレーニング**: 専門的なサポートと豊富なトレーニングプログラムを提供し、企業のITチームを支援。
- **エコシステム**: 幅広いエコシステムにより、多様なソリューションと互換性を持つ。

### 利用例
- **サーバー環境**: 企業のデータセンターやクラウド環境でのサーバーOSとして広く使用。
- **開発プラットフォーム**: 開発者向けのツールと統合され、アプリケーション開発の基盤として利用。
- **ITインフラストラクチャ**: ネットワーク、ストレージ、データベースなどのインフラストラクチャの基盤として使用。

### 結論
Red Hatは、エンタープライズ環境での高い信頼性とサポートを提供することで、企業のITインフラストラクチャの基盤として広く採用されている。特にRed Hat Enterprise Linux (RHEL)は、商用環境での標準的なLinuxディストリビューションとして認知されている。

## ミドルウェアとは？
ミドルウェアは、オペレーティングシステム（OS）とアプリケーションソフトウェアの間に位置するソフトウェアのこと。アプリケーションが効率的に動作するための共通の機能やサービスを提供する。

### ミドルウェアの役割
- **通信管理**: 異なるネットワーク間やシステム間の通信を管理し、データ交換を容易にする。
- **データ管理**: データベース接続やデータのキャッシュ管理、トランザクション管理などをサポート。
- **認証と認可**: ユーザー認証やアクセス制御を行い、セキュリティを確保。
- **メッセージング**: メッセージキューやトピックを通じて、非同期のメッセージングを提供。

### 代表的なミドルウェア
- **Webサーバ**: Apache HTTP Server、Nginx
    - クライアントからのHTTPリクエストを受け取り、適切なコンテンツを返す。
- **アプリケーションサーバ**: Tomcat、JBoss
    - Webアプリケーションを実行し、動的なコンテンツを生成する。
- **データベース管理システム（DBMS）**: MySQL、PostgreSQL
    - データの格納、検索、更新を効率的に行う。
- **メッセージキュー**: RabbitMQ、Apache Kafka
    - アプリケーション間の非同期通信を実現し、メッセージの配送を管理する。

### ミドルウェアの利用例
- **Webアプリケーション**: Webサーバとアプリケーションサーバを組み合わせて動的なWebサイトを構築。
- **エンタープライズシステム**: データベースやメッセージキューを使用して大規模な企業システムを構築。
- **クラウドサービス**: クラウド環境でのアプリケーション展開を支援するミドルウェア（例: AWS Elastic Beanstalk）。

## インスタンスタイプ
サーバで処理を実行する装置(CPU、メモリ等)を選択。インスタンスタイプの名前は`t2.micro`のように表される。

| 名称                   | 概要                                                                                 |
|------------------------|--------------------------------------------------------------------------------------|
| vCPU                   | 仮想CPUの個数                                                                         |
| アーキテクチャ         | x86_64, Arm64等のCPUアーキテクチャ                                                    |
| メモリ                 | 仮想メモリの容量                                                                      |
| ネットワーク帯域幅     | EC2外部と通信する際の帯域幅                                                           |
| ストレージ             | インスタンスストアの容量(EBSではない)                                                 |
| GPU                    | GPUの個数 (通常は0だがDeep Learning用途で必要)                                        |

## ストレージ (EBS)
EC2にもストレージが必要で、起動用OSがインストールされたストレージ (ルートボリューム) は必須。一般的にはEBS (Elastic Block Store)が使われるが、特殊用途ではインスタンスストアもある。

| 名称                   | 着脱(アタッチ)可否 | 冗長性 | データの永続性 | バックアップ(スナップショット) | 速度(スループット) | 速度(IOPS)          |
|------------------------|-------------------|--------|----------------|-------------------------------|---------------------|---------------------|
| EBS                    | 可能              | あり   | あり           | 可能                           | 高速                | SSDは高速, HDDは低速|
| インスタンスストア     | 不可              | なし   | なし           | 不可                           | EBSより高速         | EBSより超高速      |

### EBS
EBSはEC2に自由に着脱可能なストレージ。EBSもインスタンスタイプと同様に一意の名称から選択できる。

| ボリュームタイプ名称 | 種類 | 耐久性      | スループット(MB/s) | IOPS(回/s)  | 容量       | 備考                         |
|----------------------|------|-------------|---------------------|-------------|------------|------------------------------|
| gp3                  | SSD  | 99.8-99.9%  | 1000                | 16,000      | 1GB-16TB   |                              |
| gp2                  | SSD  | 99.8-99.9%  | 250                 | 16,000      | 1GB-16TB   |                              |
| io2Block Express     | SSD  | 99.8%       | 1000                | 16,000      | 4GB-64TB   |                              |
| io2                  | SSD  | 99.999%     | 4000                | 256,000     | 4GB-16TB   |                              |
| io1                  | SSD  | 99.999%     | 1000                | 64,000      | 4GB-16TB   | 複数インスタンス共有可       |
| st1                  | HDD  | 99.8-99.9%  | 500                 | 500         | 125GB-16TB | ルートボリューム不可         |
| sc1                  | HDD  | 99.8-99.9%  | 250                 | 250         | 125GB-16TB | ルートボリューム不可         |

### スナップショット
EBSのデータをS3にバックアップする機能。障害時の復旧に役立つが、有料。また、インスタンスストアでは使用できない。

## S3とは？
S3 (Simple Storage Service)は、AWSが提供するオブジェクトストレージサービス。大量のデータを安全かつ安価に保存でき、インターネット経由でアクセスが可能。

### S3の特徴
- **高い耐久性**: データは自動的に複数の施設にレプリケートされ、11 9s (99.999999999%)の耐久性を持つ。
- **スケーラビリティ**: ほぼ無限の容量を持ち、必要に応じてスケールアウトが可能。
- **セキュリティ**: IAMポリシー、バケットポリシー、ACLなどの細かいアクセス制御が可能。データはデフォルトで暗号化され、サーバー側の暗号化やクライアント側の暗号化にも対応。
- **費用対効果**: 使用した分だけの従量課金制で、初期費用や最低利用期間はない。

### S3の主要機能
- **バケット**: データを格納するためのコンテナ。各バケットはグローバルに一意の名前を持つ。
- **オブジェクト**: S3に保存される個々のファイル。オブジェクトにはメタデータとデータが含まれる。
- **バージョニング**: バケット内のオブジェクトの異なるバージョンを保持し、誤って削除や上書きした場合でも以前のバージョンに戻せる。
- **ライフサイクル管理**: オブジェクトのライフサイクルを自動的に管理し、指定したルールに基づいてオブジェクトを削除または低コストのストレージクラスに移行する。
- **イベント通知**: バケット内のオブジェクトに対する特定の操作（例: 作成、削除）をトリガーにして通知を送信する。

### 利用例
- **バックアップとリカバリー**: 重要データのバックアップ先として利用。高い耐久性と可用性を活かして災害対策にも適している。
- **データアーカイブ**: 低頻度アクセスデータの長期保存に利用。ライフサイクルポリシーを使用してコストを最適化。
- **ビッグデータ分析**: 大規模データセットを保存し、Amazon EMRやAWS Glueなどの分析ツールと連携してデータ処理を行う。
- **静的ウェブホスティング**: HTML、CSS、JavaScriptなどの静的ファイルをホスティングし、Webサイトを提供する。

### インスタンスストア
インスタンスタイプによってはインスタンスストアも選択できる。EBSに比べデータ消失のリスクがあるが、高速アクセスが可能。

### EBSとインスタンスストアの選択
- EBS: 速度はそこそこだが、データの消失対策やコスト面で優れる。一般的にはこちら。
- インスタンスストア: 超高速だが、データ消失リスクが高く、コストも高い。一時ストレージ向け。

### S3やEFSとの違い
EBSやインスタンスストアはEC2インスタンスと一緒に動作し、通常のPCでいう内蔵HDDに相当。S3は独立したサービスとして動作し、通常のPCでいうとDropBoxやGoogleドライブのような存在。EFSはEBSのようにEC2にマウントでき、複数のEC2インスタンスで共有可能。

| 動作形態                      | 冗長化方法          | データ保持形式   |
|-------------------------------|---------------------|------------------|
| EBS                           | 単一AZ内で冗長化    | ブロックストレージ |
| S3                            | 複数AZで冗長化      | オブジェクトストレージ |
| EFS                           | 複数AZで冗長化      | ファイルストレージ   |

<details><summary>AZ (Availability Zone)とは？</summary>

## AZ (Availability Zone)とは？
AZ (Availability Zone)は、AWSのデータセンターの物理的な区分。各リージョン内に複数のAZがあり、それぞれが独立した電源、ネットワーク、冷却設備を持つ。これにより、1つのAZで障害が発生しても、他のAZに影響を与えないように設計されている。

### AZの特徴
- **独立性**: 各AZは独立して運用されており、物理的な距離も一定以上離れているため、障害や災害の影響を受けにくい。
- **高可用性**: 複数のAZを利用することで、システムの可用性を高めることができる。例えば、同じリージョン内の別のAZにバックアップを配置することで、障害時のデータ損失を防げる。
- **低レイテンシ**: 同一リージョン内のAZ間は高帯域幅、低レイテンシのネットワークで接続されているため、高速なデータ転送が可能。

### 利用方法
- **冗長化**: アプリケーションやデータベースを複数のAZに分散配置することで、システムの耐障害性を向上させる。
- **バックアップ**: 定期的に別のAZにデータをバックアップし、障害時の迅速な復旧を実現する。
- **分散処理**: 高負荷の処理を複数のAZに分散させることで、負荷分散とスケーラビリティを実現する。

AZを適切に活用することで、AWS上のシステムは高い可用性と耐障害性を持つことができる。


</details></div>

## EC2とネットワーク
EC2に外部からアクセスするにはネットワークの構築が必要。

| 名称                   | 概要                                                                 |
|------------------------|----------------------------------------------------------------------|
| VPC                    | AWS上で仮想ネットワークを構築し、セキュリティや耐障害性を向上させる |
| ENI (Elastic Network Interface) | インスタンスに新たなIPアドレスを付与する                     |
| Elastic IP             | インスタンスに固定のグローバルIPアドレスを付与する                   |
| ELB (ロードバランサー); ALB | 複数のインスタンスにアクセスを分散させ、可用性向上や負荷分散を自動化する |
| Auto Scaling           | 負荷に応じてインスタンス数を自動で変化させ、コストと性能のバランスを自動調整する |
| プレイスメントグループ | 複数のインスタンスをグループ化し、インスタンス間の通信速度や耐障害性を向上させる |
| SSHによるアクセス      | キーペアで公開鍵認証によるSSH通信でインスタンスに接続し、外部からセキュアに操作する |

### メモ：SSHによるアクセス
SSH (Secure SHell)は、暗号化通信を実現するプロトコルの一つ。主にサーバーの遠隔操作に使われ、EC2でもインスタンス操作の基本的な接続手法となる。

#### SSH接続方法
PCからEC2インスタンスにアクセスするためのデフォルトツール:

- **Mac, Linux (Ubuntu)**: ターミナル
- **Windows**: コマンドプロンプトまたはPowerShell

(※2018年以降のWindows10にはOpenSSHがプリインストールされているため、デフォルトのツールでアクセス可能)

#### 注意点
SSH接続にはTCP22番ポートを使用するため、**セキュリティグループなどのファイアウォール設定でTCP22番ポートを開ける必要**がある。

<details><summary>セキュリティグループとは？</summary>

## セキュリティグループとは？
セキュリティグループは、AWSでEC2インスタンスなどのリソースに対する仮想ファイアウォールの役割を果たす。特定のトラフィックのみを許可または拒否するためのルールを設定できる。

### 特徴
- **ステートフル**: インバウンドルールで許可したトラフィックのアウトバウンドは自動的に許可される。
- **インバウンド/アウトバウンドルール**: 各セキュリティグループにはインバウンドルール（受信トラフィック）とアウトバウンドルール（送信トラフィック）を設定できる。
- **複数のリソースに適用可能**: 同じセキュリティグループを複数のリソースに適用でき、設定の管理が容易になる。
- **IPアドレスやポートの制御**: ルールは特定のIPアドレス範囲やポート番号に基づいて設定できる。

### セキュリティグループの設定例
- **インバウンドルール**: 特定のポートやIPアドレスからのトラフィックを許可するルール。
    - 例: TCPの22番ポート（SSH）を0.0.0.0/0（全てのIPアドレス）からのアクセスを許可。
- **アウトバウンドルール**: インスタンスから外部へのトラフィックを制御するルール。
    - 例: 全てのトラフィックを許可（デフォルト設定）。

### 使い方
1. **セキュリティグループの作成**:
    - AWSマネジメントコンソールで「セキュリティグループ」を選択し、新しいセキュリティグループを作成。
2. **ルールの追加**:
    - インバウンドルールとアウトバウンドルールを追加。ポート番号、プロトコル、IPアドレス範囲を指定。
3. **リソースに適用**:
    - 作成したセキュリティグループをEC2インスタンスや他のリソースに適用。

### 注意点
- **デフォルトでは全て拒否**: 新規作成時、インバウンドトラフィックは全て拒否され、アウトバウンドトラフィックは全て許可される。
- **ルールの変更は即時反映**: セキュリティグループのルールを変更すると、即座に適用される。

</details></div>
